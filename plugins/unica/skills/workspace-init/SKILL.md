# /workspace-init — первый workspace проекта

Цель: из пустого репозитория получить рабочий `v8project.yaml`, каталог исходников и проверенный путь для MCP `unica`.

## Жесткие стопы

- Если любая команда показывает проблему лицензии 1С, остановись. Не лечи лицензию, не меняй `nethasp.ini`, службу лицензирования, реестр, программную лицензию, HASP/LM и пути платформы. Сообщи пользователю, что нужно исправить лицензию на стороне окружения.
- Если база без указанного пользователя/пароля, проверь только `Администратор` без пароля, затем `Admin` без пароля. Если оба не подходят, остановись и спроси учетные данные у пользователя.
- Не перезаписывай существующий `v8project.yaml` или файл из `V8TR_CONFIG` без явного согласия.

## Workflow

1. Определи корень проекта:

```sh
git rev-parse --show-toplevel 2>/dev/null || pwd
```

2. Найди активный конфиг:
   - если задан `V8TR_CONFIG`, используй его;
   - иначе используй `./v8project.yaml`;
   - если файл существует, прочитай `connection` и `source-set`, не создавай новый.

3. Если конфига нет, получи строку соединения:
   - из аргумента пользователя;
   - иначе спроси один раз: файловая база `/F/path/to/ib` или серверная `/Sserver/ref`.

4. Создай минимальный workspace:

```sh
RUNNER=""
for p in ./plugins/unica/scripts/run-v8-runner.sh ./scripts/run-v8-runner.sh ../../scripts/run-v8-runner.sh; do
  if [ -x "$p" ]; then RUNNER="$p"; break; fi
done
test -n "$RUNNER" || { echo "Unica v8-runner launcher not found" >&2; exit 127; }
mkdir -p src
"$RUNNER" config init --file ./v8project.yaml --connection '<connection>'
```

Используй найденный `$RUNNER` во всех следующих командах.

5. Проверь доступ к базе через `/db-auth-check`. При отсутствии логина он должен попробовать только:
   - `Администратор` с пустым паролем;
   - `Admin` с пустым паролем.

6. Заполни исходники:
   - если `source-set` пустой или каталог `src` пустой, считай базу источником правды и сделай полный dump:

```sh
"$RUNNER" dump --mode full
```

   - если исходники уже есть, не загружай их в базу автоматически; сначала спроси, база или Git является источником правды.

7. Проверь готовность:

```sh
"$RUNNER" launch designer --help
test -f v8project.yaml
test -d src
```

После этого MCP `unica` может стартовать из корня проекта и сам синхронизировать workspace/cache state.

## Правила автоматизации

- Создавай `src/` автоматически, если в репозитории нет исходников.
- Для первой пустой рабочей копии предпочитай безопасный `dump` из существующей базы, а не `build` в базу.
- Не добавляй отдельные JSON/INI реестры проекта: единственный контракт — `v8project.yaml` или `V8TR_CONFIG`.
- Пароли не коммить. Если пароль нужно сохранить в connection string, сначала предупреди пользователя и предложи переменную окружения/локальный некоммитимый конфиг, если это поддерживает текущий инструмент.
